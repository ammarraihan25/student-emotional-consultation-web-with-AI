================================================================================
          PANDUAN PENGGUNAAN DAN DOKUMENTASI ARSITEKTUR FILE AETHER
       Sistem Konsultasi Emosional Mahasiswa Berbasis AI & NLP Engine
================================================================================

Selamat datang di Aether! Dokumen ini dibuat untuk memandu Anda tentang cara 
menjalankan aplikasi, alur kerja sistem, serta fungsi dari setiap file penting 
yang ada di dalam seluruh repositori ini.

Aether adalah platform kesehatan mental modern bertema "Calm Sage & Botanical"
yang dirancang khusus bagi mahasiswa untuk mengenali dan mengonsultasikan kondisi
emosional mereka secara privat, interaktif, dan penuh empati.

--------------------------------------------------------------------------------
1. ARSITEKTUR SISTEM (3-TIER ARCHITECTURE)
--------------------------------------------------------------------------------
Sistem Aether dibagi menjadi 3 bagian utama yang saling terintegrasi:

1. FRONTEND CLIENT (aether-frontend)
   - Teknologi: Vue 3 (Composition API), Vite, Tailwind CSS v4.0.0.
   - Fungsi: Menyediakan antarmuka pengguna yang modern, premium, responsif, 
     dan menenangkan (Zen theme) dengan visualisasi bagan grafik suasana hati,
     percakapan real-time, serta modul pusat edukasi mandiri.

2. BACKEND API SERVER (web-konseling)
   - Teknologi: Laravel 11, PHP, Laravel Sanctum, MySQL Database.
   - Fungsi: Mengatur manajemen pengguna (autentikasi), menyimpan riwayat sesi 
     konsultasi, serta menyajikan data statistik mingguan mood tracking.

3. AI & NLP SCORING ENGINE (ai-konseling)
   - Teknologi: FastAPI (Python 3), Uvicorn, NLTK, Google Gemini AI API.
   - Fungsi: Memproses curahan hati mahasiswa melalui NLP berbasis Rule-Based 
     Scoring untuk menghasilkan skor tingkat distress (0-100) dan klasifikasi 
     emosi (Hijau/Kuning/Merah). Kemudian menggunakan Gemini AI untuk 
     membuat respons balasan yang empatik berbasis teknik psikologis CBT 
     (Cognitive Behavioral Therapy).

--------------------------------------------------------------------------------
2. PERSYARATAN & CARA MENJALANKAN SISTEM
--------------------------------------------------------------------------------

A. Prasyarat Sistem:
   - Laragon atau XAMPP (dengan MySQL aktif).
   - PHP >= 8.2 dan Composer.
   - Node.js >= 18 dan npm.
   - Python >= 3.10 dan pip.

B. Langkah Awal (Database):
   1. Pastikan Laragon/MySQL Anda menyala.
   2. Buat database kosong bernama `db_konseling` di phpMyAdmin (atau MySQL CLI).
   3. Konfigurasi file `.env` di dalam direktori `web-konseling` agar mengarah ke 
      database tersebut (DB_DATABASE=db_konseling, DB_USERNAME=root, DB_PASSWORD=).
   4. Jalankan perintah migrasi di command prompt pada folder `web-konseling`:
      > php artisan migrate

C. Cara Menjalankan Server (3 Terminal Berbeda):

   TERMINAL 1: Laravel Backend Server (Port 8080)
   1. Buka folder `web-konseling`.
   2. Jalankan perintah:
      > php artisan serve --port=8080

   TERMINAL 2: FastAPI AI Server (Port 8000)
   1. Buka folder `ai-konseling`.
   2. Aktifkan virtual environment Python:
      > .\venv\Scripts\Activate.ps1
   3. Jalankan perintah:
      > uvicorn main:app --reload --port 8000

   TERMINAL 3: Vue 3 Frontend Client (Port 5173 / 5175)
   1. Buka folder `aether-frontend`.
   2. Pastikan dependencies terinstal:
      > npm install
   3. Jalankan server dev:
      > npm run dev

   *Catatan: Secara bawaan, Vite akan mendeteksi port yang tersedia. Buka alamat 
   URL yang ditampilkan (biasanya http://localhost:5173 atau http://localhost:5175) 
   di browser Anda untuk mengakses aplikasi.*

3. FUNGSI PER FILE & DIREKTORI
--------------------------------------------------------------------------------

================================================================================
A. MODUL FRONTEND CLIENT (`aether-frontend`)
================================================================================
Direktori ini berisi kode antarmuka Vue 3 yang ramah pengguna.

- `package.json`
  Mendefinisikan daftar pustaka (dependencies) frontend seperti Vue 3, Pinia, 
  Vue Router, Tailwind CSS, dan Lucide Icons.
  
- `vite.config.js`
  Konfigurasi build tools Vite, menyertakan plugin Vue dan Tailwind CSS, serta 
  mengatur server Proxy. Rute `/api` diarahkan ke Laravel (port 8080) dan `/ai` 
  diarahkan ke FastAPI (port 8000) untuk menghindari masalah CORS.

- `src/main.js`
  File entri utama Javascript untuk inisialisasi aplikasi Vue, menyambungkan 
  Pinia (state management) dan Vue Router, serta memuat file gaya global.

- `src/style.css`
  Pusat desain dan tema aplikasi. Di sinilah **Calm Sage & Botanical Dark Theme** 
  dikonfigurasi menggunakan Tailwind CSS v4 `@theme`. Mengatur scrollbar hijau sage, 
  gradien teks bertema alam, efek glassmorphic lembut kehijauan (`.glass`), 
  tombol berpijar (`.btn-glow`), serta berbagai animasi penenang pikiran.

- `src/App.vue`
  Komponen root utama yang merender halaman aktif sesuai rute saat ini dengan 
  transisi halus halaman (`<transition>`).

- `src/api/auth.js`
  Berisi fungsi pemanggilan HTTP (menggunakan axios) ke Laravel Backend untuk 
  registrasi, login, demo login cepat, dan logout.

- `src/api/chat.js`
  Berisi fungsi pemanggilan HTTP ke Laravel untuk memuat daftar sesi percakapan,
  menyimpan percakapan, mengirim pesan curhat mahasiswa untuk dianalisis oleh AI, 
  dan menghapus riwayat sesi.

- `src/stores/auth.js`
  Pinia Store untuk menyimpan informasi login pengguna secara global dan token 
  otentikasi (JWT) yang persisten di localStorage. Menyediakan fitur Demo Login.

- `src/stores/chat.js`
  Pinia Store untuk menyimpan daftar percakapan aktif, memproses pengiriman pesan 
  dan respons AI secara real-time, serta menghitung status klasifikasi emosional 
  saat percakapan berlangsung.

- `src/stores/mood.js`
  Pinia Store untuk memuat data statistik mingguan mood tracking dari database 
  Laravel, guna divisualisasikan kepada mahasiswa.

- `src/router/index.js`
  Pengatur navigasi rute klien. Mengamankan halaman khusus login (seperti /chat, 
  /mood, /resources) menggunakan Navigation Guards agar hanya bisa diakses oleh 
  pengguna yang sudah login/terotentikasi.

- `src/components/layout/Navbar.vue`
  Bilah navigasi atas di Landing Page untuk berpindah halaman dan tombol login.

- `src/components/layout/Sidebar.vue`
  Bilah samping interaktif pada halaman dashboard. Menampilkan daftar riwayat 
  sesi percakapan, status user, menu navigasi, dan tombol logout.

- `src/components/ui/EmotionBadge.vue`
  Komponen visual lencana indikator emosional mahasiswa (🟢 Stabil / 🟡 Distress 
  / 🔴 Krisis) lengkap dengan nilai skor NLP-nya.

- `src/components/ui/FeatureCard.vue`
  Kartu grid interaktif di halaman Landing Page untuk menampilkan fitur unggulan Aether.

- `src/components/ui/ResourceCard.vue`
  Kartu modul edukasi mandiri pada Resource Center yang berisi tip/meditasi.

- `src/components/mood/MoodChart.vue`
  Grafik garis interaktif (menggunakan grafik SVG ringan) untuk melacak tren naik-turun 
  skor tingkat stres mahasiswa dalam 7 hari terakhir.

- `src/views/LandingPage.vue`
  Halaman depan portal Aether. Berisi ilustrasi, penjelasan sistem, alur cara kerja, 
  dan tombol ajakan (CTA) untuk mulai berbicara dengan AI.

- `src/views/LoginPage.vue` & `src/views/RegisterPage.vue`
  Halaman masuk dan pendaftaran akun baru dengan desain kartu kaca kehijauan yang rapi.

- `src/views/ChatPage.vue`
  Inti aplikasi. Tempat mahasiswa bercakap-cakap dengan Aether AI. Menampilkan chat 
  bubble interaktif, mengetik otomatis (typewriter), indikator analisis emosi real-time, 
  dan panel panduan/rekomendasi tindakan mandiri.

- `src/views/MoodTracking.vue`
  Dasbor analitik emosional mahasiswa. Menyajikan visualisasi rata-rata skor stres, 
  hari berturut-turut bercerita (streak), diagram distribusi mood (stabilitas emosi), 
  dan riwayat kalender mingguan.

- `src/views/ResourceCenter.vue`
  Pusat bantuan mandiri. Mahasiswa dapat memfilter bahan bacaan/panduan seperti latihan 
  pernapasan 4-7-8, tips menghadapi dosen, journaling terapeutik, dan meditasi pagi.

- `src/views/NotFound.vue`
  Halaman error 404 estetik bertema botani 🌿 jika mahasiswa mengakses rute yang salah.


================================================================================
B. MODUL BACKEND API SERVER (`web-konseling`)
================================================================================
Direktori Laravel yang berfungsi sebagai motor manajemen basis data dan otentikasi.

- `routes/api.php`
  Mendefinisikan endpoint RESTful API yang aman menggunakan middleware Laravel Sanctum. 
  Menyediakan endpoint registrasi, masuk, keluar, riwayat obrolan, simpan obrolan, 
  dan kalkulasi statistik mingguan.

- `app/Http/Controllers/AuthController.php`
  Mengontrol registrasi akun baru, otentikasi kecocokan password, pembuatan token 
  keamanan JWT, serta menangani otentikasi Demo Login khusus tanpa registrasi.

- `app/Http/Controllers/ChatController.php`
  Mengontrol penyimpanan pesan, memuat riwayat obrolan berdasarkan user aktif, 
  menghapus sesi chat, dan mengalkulasi metrik statistik krusial (seperti rata-rata 
  skor stres, tingkat stabilitas emosi, streak curhat, dan distribusi mingguan).

- `app/Models/User.php`
  Model data pengguna yang terotentikasi di MySQL, bersambung ke sesi percakapan.

- `app/Models/ChatSession.php`
  Model data sesi konsultasi emosional. Menyimpan referensi `user_id`, `title` sesi, 
  `max_score` stres yang terdeteksi, status `final_emotion` (Hijau/Kuning/Merah), 
  serta seluruh riwayat pertukaran pesan dalam format JSON (`messages`).

- `database/migrations/`
  Skrip migrasi SQL untuk membuat tabel `users` dan tabel `chat_sessions` beserta 
  kolom data tipe yang dibutuhkan secara otomatis di database MySQL.


================================================================================
C. MODUL AI & NLP SCORING ENGINE (`ai-konseling`)
================================================================================
Direktori Python yang bertindak sebagai mesin pintar penganalisis emosi dan respons AI.

- `main.py`
  Server utama FastAPI. Menyediakan endpoint POST `/analyze` yang dipanggil oleh 
  frontend saat mahasiswa berkirim pesan curhat. Endpoint ini akan:
  1. Menerima teks pesan dari mahasiswa.
  2. Menganalisis teks menggunakan modul `nlp_engine.py`.
  3. Memilih respons cepat jika pesan cocok dengan kasus uji standar di 
     `predefined_responses.py`.
  4. Jika tidak cocok dengan data predefined, server akan memanggil Google Gemini 
     AI dengan prompt khusus yang mengajarkan Gemini bertindak sebagai psikolog empati 
     berbasis Cognitive Behavioral Therapy (CBT).
  5. Mengembalikan hasil analisis emosi lengkap beserta skor stres dan teks balasan AI.

- `nlp_engine.py`
  Algoritma inti pemrosesan bahasa alami (NLP) tradisional berkinerja tinggi. Melakukan:
  - Normalisasi teks (mengubah slang/singkatan mahasiswa Indonesia seperti 'gpp' -> 'tidak 
    apa-apa', 'insecure' -> 'cemas merasa rendah diri').
  - Tokenisasi kata dan pencocokan pola kata kunci (keyword matching).
  - Skoring berbasis Rule-Based Scoring pada 4 kategori parameter emosi:
    1. Kategori Masalah Akademik/Sosial (skor 0-40)
    2. Kategori Urgensi Emosional (skor 0-30)
    3. Kategori Risiko Krisis/Bahaya Diri (skor 0-20)
    4. Kategori Regulasi Emosional Positif (skor pengurang hingga -20)
  - Klasifikasi status emosi:
    🟢 Skor 0 - 35   : STABIL (Hijau)
    🟡 Skor 36 - 70  : DISTRESS EMOSIONAL (Kuning)
    🔴 Skor > 70     : KRISIS (Merah)

- `predefined_responses.py`
  Kumpulan 170+ pola kasus uji curhatan mahasiswa dan respons terapeutik standar. 
  Berguna untuk memastikan keakuratan klasifikasi 100% dan respon secepat kilat (sub-milidetik)
  untuk percakapan umum yang sering ditanyakan mahasiswa.

- `cek_model.py` & `test_nlp.py`
  Skrip pembantu developer untuk menguji langsung API Gemini dan keakuratan mesin NLP 
  tanpa harus membuka UI website.

--------------------------------------------------------------------------------
4. ALUR KERJA PLATFORM (BAGAIMANA SISTEM BEKERJA)
--------------------------------------------------------------------------------

1. Mahasiswa membuka portal Aether, melakukan registrasi, atau masuk menggunakan 
   fitur Demo Login Cepat.
2. Mahasiswa masuk ke halaman Chat, memulai sesi baru, lalu mengetikkan curahan hati 
   mereka (contoh: "Aku stres banget sama tugas kuliah, rasanya burnout dan mau menyerah").
3. Pesan dikirim dari browser (frontend) ke server Laravel `/api/chat/message`.
4. Laravel meneruskan pesan tersebut ke FastAPI AI Server di `/analyze`.
5. FastAPI memproses pesan tersebut menggunakan `nlp_engine.py`:
   - Slang dinormalisasi.
   - Kata kunci "stres", "burnout", "menyerah" terdeteksi dan dihitung skornya.
   - Didapatkan skor Stres (misal: 42) yang secara otomatis diklasifikasikan ke 
     dalam status distress emosional (🟡 Kuning).
   - Mesin AI (Gemini CBT / Predefined) memformulasikan respons psikologis yang sangat 
     lembut, berempati, dan memberikan solusi relaksasi yang menenangkan.
6. Hasil analisis dan teks balasan AI dikembalikan ke Laravel, lalu disimpan ke dalam 
   database `db_konseling` pada tabel `chat_sessions`.
7. Browser (frontend) menerima data, memperbarui antarmuka chat, menampilkan gelembung 
   obrolan AI, memperbarui lencana indikator emosi di pojok atas secara dinamis, serta 
   menghidupkan visualisasi di modul Mood Tracker.
